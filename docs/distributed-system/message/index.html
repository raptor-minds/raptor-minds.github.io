Transaction#

The processing flow of RocketMQ in the distributed system can be generally descriped as followesï¼š

Two phase tx commit#

  1. Producer need to invoke A,B,C,D method in one Tx, the broker of rocket message queue will store these four message in one Half Queue.

  2. The broker will start one thread to schedully call back the execution status of A, B,C, D. When the status is OK, then the broker will put the A,B,C,D message in the Final Queue.
